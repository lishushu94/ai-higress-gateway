### 删除标准

删除的文件满足以下条件之一：

1. **过时的文档** - 信息已被新文档替代
2. **不完整的文档** - 信息不完整或不准确
3. **重复的文档** - 内容与其他文档重复
4. **临时文件** - 构建过程中生成的日志文件
5. **已弃用的文档** - 不再适用于当前项目

### 保留标准

保留的文件满足以下条件：

1. **最新的文档** - 包含最新的信息和最佳实践
2. **完整的文档** - 信息完整、准确、有用
3. **独特的文档** - 提供独特的价值，不与其他文档重复
4. **实用的文档** - 对开发者有实际帮助
