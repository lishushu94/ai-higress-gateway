# AI Higress 前端页面设计提示词

## 核心要求

请为AI Higress智能路由网关系统设计一个**简洁而优雅**的前端管理界面。设计风格应该遵循"少即是多"的原则，避免过度装饰，但保持专业和现代感。

## 设计风格指导

### 整体美学
- **极简主义**：使用最少的元素实现最大功能
- **东方美学**：融入中国传统美学的留白和平衡感
- **现代专业**：符合当代企业级应用的设计趋势
- **墨水风格**：采用类似中国水墨画的视觉效果，黑白灰为主色调，点缀少量强调色

### 色彩方案
- **主色调**：深灰色(#1a1a1a)和纯白(#ffffff)
- **辅助色**：浅灰色(#f5f5f5)和中灰色(#d0d0d0)
- **强调色**：深蓝(#0066cc)或暗红(#993333)少量点缀
- **状态色**：成功(#4CAF50)、警告(#FFC107)、错误(#F44336)

### 字体选择
- **主字体**：系统无衬线字体(San Francisco, Segoe UI, PingFang SC)
- **标题字体**：细字重的无衬线字体
- **代码字体**：等宽字体(Consolas, Monaco, Source Code Pro)

## 页面布局原则

### 网格系统
- 使用12列网格系统
- 保持大量留白，元素间距至少24px
- 内容区域最大宽度1200px，居中对齐

### 导航设计
- **简洁顶部导航**：仅保留必要的导航项和用户信息
- **侧边栏导航**：采用图标+文字的极简设计
- **面包屑导航**：细线样式，不抢夺视觉焦点

### 卡片设计
- **轻量卡片**：细边框或无边框，通过微妙的阴影区分
- **卡片间距**：统一24px垂直间距
- **卡片内部**：保持足够的内部间距(20px)

## 组件设计规范

### 按钮
- **主要按钮**：实心填充，圆角4px，高度40px
- **次要按钮**：线条样式，同规格
- **悬停效果**：轻微阴影和色值变化，避免夸张效果

### 表单元素
- **输入框**：无边框设计，底部细线作为分隔
- **标签**：小号字，放置在输入框内部或上方
- **验证状态**：使用细线颜色变化，避免夸张的错误提示

### 数据表格
- **极简表格**：细线边框或无线条，通过行间距区分
- **表头**：小号字，全部大写，增加字间距
- **行高**：48px，提供足够的呼吸空间

### 图表
- **简洁图表**：使用最少的网格线、图例和标签
- **单色图表**：使用同色系的不同深浅表示数据
- **平滑曲线**：避免锯齿，使用平滑过渡

## 页面特定设计要求

### 仪表盘页面
- **关键指标**：大号数字，细线框架，极简标注
- **状态卡片**：最小化的状态指示，避免过多装饰
- **数据可视化**：使用简洁的线条图，非必要时去除渐变和动画

### 提供商管理
- **提供商卡片**：仅显示核心信息，隐藏非必要细节
- **状态指示**：使用简单的点或线，避免复杂图标
- **操作按钮**：悬停时才显示，保持界面整洁

### API密钥管理
- **密钥列表**：表格形式，仅显示必要信息
- **创建表单**：步骤式设计，减少视觉负担
- **权限设置**：使用开关和标签，直观简洁

### 系统监控
- **指标图表**：大尺寸，简单配色，突出数据本身
- **时间选择器**：极简设计，使用预设时间范围
- **警报信息**：简洁的列表，仅显示关键信息

## 交互设计

### 微交互
- **悬停状态**：微妙的变化，避免夸张效果
- **点击反馈**：轻微的颜色变化或缩放
- **加载状态**：简单的加载指示器，避免复杂动画

### 过渡效果
- **页面切换**：淡入淡出，300ms
- **模态框**：从中心展开，200ms
- **下拉菜单**：平滑展开，150ms

## 技术实现建议

### CSS方法
- 使用CSS变量管理颜色和间距
- 采用Flexbox和Grid布局
- 避免过度嵌套的选择器

### 组件设计
- 单一职责原则，每个组件只做一件事
- 使用组合而非继承
- 保持组件接口简洁

### 性能考虑
- 避免不必要的重绘和重排
- 使用CSS transform实现动画
- 懒加载非关键资源

## 避免的设计陷阱

### 过度装饰
- 不要使用渐变、阴影和纹理
- 避免不必要的图标和插图
- 不要添加花哨的背景图案

### 信息过载
- 每个页面最多3-4个主要区域
- 每个卡片最多5-7个信息项
- 使用折叠和渐进式展示

### 复杂交互
- 避免多级菜单和复杂的手势
- 不使用模态框层级嵌套
- 保持简单的用户路径

## 性能优化指南

### 前端性能优化原则

在设计和实现前端界面时，应该遵循以下性能优化原则：

#### 1. 服务端优先架构

- **页面组件默认为服务端组件**：利用 Next.js App Router 的服务端渲染能力，减少客户端 JavaScript
- **数据预取**：在服务端预取初始数据，避免客户端加载时的数据请求延迟
- **客户端组件隔离**：仅在需要交互时使用客户端组件，保持页面组件简洁

#### 2. 代码分割与动态导入

- **大型组件动态导入**：使用 `next/dynamic` 对图表、对话框等大型组件进行动态导入
- **按需加载**：非首屏必需的功能应该延迟加载，减少初始 bundle 大小
- **加载占位符**：为动态导入的组件提供 Loading Skeleton，改善用户体验

#### 3. 列表与表格优化

- **虚拟滚动**：对于超过 50 个列表项的组件，使用虚拟滚动库（如 @tanstack/react-virtual）
- **分页加载**：大数据集应该使用分页或无限滚动，避免一次性渲染过多 DOM 节点
- **搜索和筛选**：提供搜索和筛选功能，让用户快速定位所需数据

#### 4. 图片优化

- **使用 Next.js Image 组件**：自动优化图片大小、格式和加载策略
- **懒加载**：非首屏图片应该设置 `loading="lazy"`
- **响应式图片**：根据设备宽度提供合适的图片尺寸

#### 5. 数据获取优化

- **SWR 缓存策略**：根据数据更新频率选择合适的缓存策略
  - `static`：很少变化的数据（如配置）
  - `frequent`：频繁更新的数据（如指标）
  - `realtime`：实时数据（如活动日志）
- **请求去重**：使用相同的 SWR key 自动合并重复请求
- **避免过度请求**：配置合适的 `revalidateOnFocus` 和 `dedupingInterval`

#### 6. 渲染优化

- **React.memo**：对纯展示组件使用 memo 避免不必要的重渲染
- **useMemo 和 useCallback**：缓存计算结果和回调函数，避免重复计算
- **组件拆分**：将大型组件拆分为小型子组件，提升渲染效率

#### 7. 构建优化

- **代码分割**：合理配置 webpack 分割策略，生成合适大小的 chunk
- **Tree shaking**：移除未使用的代码，减少 bundle 大小
- **压缩和混淆**：启用生产环境的代码压缩和混淆

### 性能指标目标

在实现前端界面时，应该追求以下性能指标：

| 指标 | 目标值 | 说明 |
|------|--------|------|
| FCP (首次内容绘制) | < 1.8s | 用户首次看到页面内容的时间 |
| LCP (最大内容绘制) | < 2.5s | 页面主要内容加载完成的时间 |
| CLS (累积布局偏移) | < 0.1 | 页面布局稳定性 |
| TTI (可交互时间) | < 3.8s | 页面可以响应用户交互的时间 |
| 初始 Bundle 大小 | < 200KB | 首屏加载的 JavaScript 大小 |
| 首屏加载时间 | < 3s | 从请求到首屏完全加载的时间 |

### 性能检查清单

在完成页面设计和实现后，应该检查以下项目：

- [ ] 页面使用服务端组件优先架构
- [ ] 大型组件已使用动态导入
- [ ] 长列表已实现虚拟滚动或分页
- [ ] 所有图片已使用 Next.js Image 组件
- [ ] 数据获取已配置合适的 SWR 缓存策略
- [ ] 纯展示组件已使用 React.memo 优化
- [ ] 组件大小不超过 200 行代码
- [ ] 已运行 Lighthouse 审计，性能分数 > 90
- [ ] 首屏加载时间 < 3 秒
- [ ] 初始 Bundle 大小 < 200KB

## 最终目标

创建一个用户感觉"呼吸顺畅"的管理界面，每个元素都有明确目的，没有多余装饰，让用户能够专注于数据和任务，而不是界面本身。界面应该像一块透明玻璃，让内容自然呈现，而不是阻挡视线的障碍。

同时，通过遵循性能优化指南，确保界面不仅美观，而且快速响应，为用户提供流畅的使用体验。

请基于以上指导原则，为AI Higress系统设计一套既简洁又优雅，又高效快速的前端界面。
